# Plan: Judge Database & Web Scraper

## Context

We're building a judge/organizer directory for the Mirrorless hackathon project (TreeHacks). The goal is to scrape biographical data and a photo for ~50 judges and organizers from the web, store it in Neon Postgres, and expose it via REST endpoints. This data can be used on the mirror display or phone UI to show who's judging.

## Step 1: Database Migration (`004_create_judges.sql`)

```sql
CREATE TABLE judges (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name          text NOT NULL,
  role          text NOT NULL DEFAULT 'judge'
                  CHECK (role IN ('judge', 'organizer')),
  title         text,              -- "CEO", "Software Engineer", etc.
  organization  text,              -- "Google", "Stanford", etc.
  bio           text,              -- 2-3 sentence bio
  photo_url     text,              -- URL to best headshot found
  linkedin_url  text,
  twitter_url   text,
  website_url   text,
  source_urls   jsonb DEFAULT '[]',  -- URLs where data was found
  scrape_status text NOT NULL DEFAULT 'pending'
                  CHECK (scrape_status IN ('pending', 'scraped', 'failed', 'manual')),
  scrape_error  text,
  scraped_at    timestamptz,
  created_at    timestamptz NOT NULL DEFAULT now(),
  updated_at    timestamptz NOT NULL DEFAULT now()
);

CREATE UNIQUE INDEX idx_judges_name_role ON judges(lower(name), role);
CREATE INDEX idx_judges_role ON judges(role);
CREATE INDEX idx_judges_scrape_status ON judges(scrape_status);
```

## Step 2: Database Operations (`judges/db.py`)

Follow the pattern from `backend/scraper/db.py` — each function takes a `db` (NeonHTTPClient) parameter with `$1, $2` parameterized queries.

Functions:
- **`seed_judges(db, judges_list)`** — Bulk insert names with `ON CONFLICT DO NOTHING` (idempotent)
- **`get_pending_judges(db)`** — Fetch judges with `scrape_status = 'pending'`
- **`update_judge_data(db, judge_id, data)`** — Update with scraped data, set status to `'scraped'`
- **`mark_judge_failed(db, judge_id, error)`** — Set status to `'failed'` with error message
- **`get_all_judges(db, role=None)`** — List all, optionally filtered by role
- **`get_judge_by_id(db, judge_id)`** — Single lookup by UUID
- **`manual_update_judge(db, judge_id, updates)`** — For PUT endpoint, sets status to `'manual'`

## Step 3: Web Scraper (`judges/scraper.py`)

### Hardcoded Judge List
The full list of 50+ names with their roles embedded as a constant `JUDGES_LIST`.

### Scraping Pipeline (per judge)

1. **Serper Web Search** → `POST https://google.serper.dev/search`
   - Query: `"{name}" TreeHacks Stanford 2025 judge` (or `organizer`)
   - For ambiguous names ("Ben", "Brandon"): `"{name}" TreeHacks 2025`
   - Returns top 5 organic results (title, snippet, link)

2. **Serper Image Search** → `POST https://google.serper.dev/images`
   - Query: `"{name}" TreeHacks headshot`
   - Returns top 5 image results (imageUrl, title)

3. **Claude Haiku Extraction** — same pattern as `purchase_parser.py:_extract_with_llm`
   - Feed web + image results into a prompt
   - Ask for JSON: `{title, organization, bio, best_photo_url, linkedin_url, twitter_url, website_url, confidence, source_urls}`
   - `max_tokens=512`, model `claude-haiku-4-5-20251001`

### Rate Limiting
- `asyncio.Semaphore(2)` for concurrent Serper calls
- 1-second sleep between batches of 3 judges
- ~100 Serper calls total (well within 2,500/month free tier)
- ~$0.02 total Claude API cost

### Ambiguous Name Handling
Names with no last name ("Ben", "Brandon", "Dj Isaac") get:
- "TreeHacks" appended to the name as context (e.g., `"Ben TreeHacks"`)
- Same search pipeline as other judges, just with this extra qualifier
- Claude returns `confidence: "low"` if results are still unclear
- Low-confidence entries stored with note in `scrape_error` for manual review

## Step 4: API Routes (`judges/routes.py`)

| Method | Path | Purpose |
|--------|------|---------|
| POST | `/judges/scrape` | Trigger scraping (background task via `asyncio.create_task`) |
| GET | `/judges/status` | Scrape progress: `{total, pending, scraped, failed}` |
| GET | `/judges` | List all judges (optional `?role=judge\|organizer` filter) |
| GET | `/judges/{judge_id}` | Single judge by UUID |
| PUT | `/judges/{judge_id}` | Manual update/correction |

**Note:** `/judges/status` must be defined before `/judges/{judge_id}` in the router to avoid FastAPI treating "status" as a UUID.

### Pydantic Models (inline in routes.py, same pattern as `scraper/routes.py`)
- `ScrapeJudgesRequest` — optional `names: list[str]`, `retry_failed: bool`
- `JudgeUpdateRequest` — all optional fields for manual edits
- `JudgeResponse` — full judge record

## Step 5: Register Router in `main.py`

Add two lines to `backend/main.py`:
```python
from judges.routes import router as judges_router
app.include_router(judges_router)
```

## Step 6: Run Migration

Execute `004_create_judges.sql` against Neon via the Neon MCP `run_sql` tool.

## Step 1: Database Migration (`004_create_judges.sql`)

```sql
CREATE TABLE judges (
  id            uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  name          text NOT NULL,
  role          text NOT NULL DEFAULT 'judge'
                  CHECK (role IN ('judge', 'organizer')),
  title         text,              -- "CEO", "Software Engineer", etc.
  organization  text,              -- "Google", "Stanford", etc.
  bio           text,              -- 2-3 sentence bio
  photo_url     text,              -- URL to best headshot found
  linkedin_url  text,
  twitter_url   text,
  website_url   text,
  source_urls   jsonb DEFAULT '[]',  -- URLs where data was found
  scrape_status text NOT NULL DEFAULT 'pending'
                  CHECK (scrape_status IN ('pending', 'scraped', 'failed', 'manual')),
  scrape_error  text,
  scraped_at    timestamptz,
  created_at    timestamptz NOT NULL DEFAULT now(),
  updated_at    timestamptz NOT NULL DEFAULT now()
);

CREATE UNIQUE INDEX idx_judges_name_role ON judges(lower(name), role);
CREATE INDEX idx_judges_role ON judges(role);
CREATE INDEX idx_judges_scrape_status ON judges(scrape_status);
```

## Step 2: Database Operations (`judges/db.py`)

Follow the pattern from `backend/scraper/db.py` — each function takes a `db` (NeonHTTPClient) parameter with `$1, $2` parameterized queries.

Functions:
- **`seed_judges(db, judges_list)`** — Bulk insert names with `ON CONFLICT DO NOTHING` (idempotent)
- **`get_pending_judges(db)`** — Fetch judges with `scrape_status = 'pending'`
- **`update_judge_data(db, judge_id, data)`** — Update with scraped data, set status to `'scraped'`
- **`mark_judge_failed(db, judge_id, error)`** — Set status to `'failed'` with error message
- **`get_all_judges(db, role=None)`** — List all, optionally filtered by role
- **`get_judge_by_id(db, judge_id)`** — Single lookup by UUID
- **`manual_update_judge(db, judge_id, updates)`** — For PUT endpoint, sets status to `'manual'`

## Step 3: Web Scraper (`judges/scraper.py`)

### Hardcoded Judge List
The full list of 50+ names with their roles embedded as a constant `JUDGES_LIST`.

### Scraping Pipeline (per judge)

1. **Serper Web Search** → `POST https://google.serper.dev/search`
   - Query: `"{name}" TreeHacks Stanford 2025 judge` (or `organizer`)
   - For ambiguous names ("Ben", "Brandon"): `"{name}" TreeHacks 2025`
   - Returns top 5 organic results (title, snippet, link)

2. **Serper Image Search** → `POST https://google.serper.dev/images`
   - Query: `"{name}" TreeHacks headshot`
   - Returns top 5 image results (imageUrl, title)

3. **Claude Haiku Extraction** — same pattern as `purchase_parser.py:_extract_with_llm`
   - Feed web + image results into a prompt
   - Ask for JSON: `{title, organization, bio, best_photo_url, linkedin_url, twitter_url, website_url, confidence, source_urls}`
   - `max_tokens=512`, model `claude-haiku-4-5-20251001`

### Rate Limiting
- `asyncio.Semaphore(2)` for concurrent Serper calls
- 1-second sleep between batches of 3 judges
- ~100 Serper calls total (well within 2,500/month free tier)
- ~$0.02 total Claude API cost

### Ambiguous Name Handling
Names with no last name ("Ben", "Brandon", "Dj Isaac") get:
- "TreeHacks" appended to the name as context (e.g., `"Ben TreeHacks"`)
- Same search pipeline as other judges, just with this extra qualifier
- Claude returns `confidence: "low"` if results are still unclear
- Low-confidence entries stored with note in `scrape_error` for manual review

## Step 4: API Routes (`judges/routes.py`)

| Method | Path | Purpose |
|--------|------|---------|
| POST | `/judges/scrape` | Trigger scraping (background task via `asyncio.create_task`) |
| GET | `/judges/status` | Scrape progress: `{total, pending, scraped, failed}` |
| GET | `/judges` | List all judges (optional `?role=judge\|organizer` filter) |
| GET | `/judges/{judge_id}` | Single judge by UUID |
| PUT | `/judges/{judge_id}` | Manual update/correction |

**Note:** `/judges/status` must be defined before `/judges/{judge_id}` in the router to avoid FastAPI treating "status" as a UUID.
