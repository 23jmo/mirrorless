# Per-Email Streaming + Full History Scraping

## Context

The scraping pipeline currently has two UX problems:
1. **No real-time feedback** ŌĆö `fast_scrape` blocks for ~15s then returns all results at once. Socket events (`emit_scrape_progress`, `emit_scrape_complete`) exist but are **never called**.
2. **Limited date range** ŌĆö `fast_scrape` only queries the last 6 months (`newer_than:6m`). Deep scrape covers full history but runs silently in the background with no visibility.

**Goal**: Stream each parsed email to the frontend as it's found (via Socket.io), and remove date limits so we scrape full history in one pass.

## Architecture Change

**Before**: POST `/api/scrape/start` ŌåÆ blocks ~15s ŌåÆ returns all results ŌåÆ fires background deep_scrape (invisible)
**After**: POST `/api/scrape/start` ŌåÆ returns immediately `{status: "started"}` ŌåÆ entire scrape runs as background task ŌåÆ streams per-email Socket.io events ŌåÆ emits `scrape_complete` with final profile

This means the frontend switches from HTTP-response-driven to fully Socket.io-driven for results.

## Step 3: Add `emit_purchase_found` socket event (`socket_events.py`)

New event that fires for each email parsed:

```python
async def emit_purchase_found(sio, user_id: str, email_subject: str, purchases: list[dict], total_so_far: int):
    await sio.emit("purchase_found", {
        "user_id": user_id,
        "email_subject": email_subject,
        "purchases": purchases,
        "total_so_far": total_so_far,
    }, room=user_id)
```

## Step 4: Rewire routes (`routes.py`)

Change POST `/api/scrape/start` to:
1. Validate token exists
2. Kick off `_background_scrape(user_id, token_data, sio)` as asyncio task
3. Return immediately: `{"status": "started"}`

The background task:
- Gets `sio` from `app.state.sio` (already available ŌĆö set in `main.py`)
- Emits `scrape_progress` with `phase: "searching"` at start
- Passes `on_email` callback that emits `purchase_found` for each parsed email
- Stores purchases to DB after each email (not in bulk at the end)
- Emits `scrape_complete` with the final profile when done
- Emits `scrape_progress` with `phase: "error"` if something fails

Access `sio` via request: add `request: Request` param to the route handler, then `request.app.state.sio`.
