# Plan: Wire Up Full Onboarding + Mirror Kiosk + Admin Panel

## Context

The phone onboarding page uses hardcoded UUIDs and a manual MVP flow. All building blocks exist (OAuth, scraping, queue) but aren't wired together. This plan implements the full end-to-end flow for a **public event booth**: QR scan → phone onboarding → queue → mirror kiosk → session recap. Full spec at `ONBOARDING_SPEC.md`.

## Scope Summary

Three workstreams:
1. **Phone onboarding** — `signin → questionnaire → queue → idle → recap` with real Google OAuth, selfie capture, background scraping, returning user detection
2. **Mirror kiosk** — continuous operation: `attract (QR) → waiting (Start/Skip) → session → cycle`. Auto-advances queue. Graceful degradation when scrape is incomplete.
3. **Admin panel** — `/admin` route: queue list with reorder/skip, current session controls, booth stats

## Changes by File

### Backend

#### `backend/migrations/009_add_selfie_column.sql` (NEW)
- `ALTER TABLE users ADD COLUMN IF NOT EXISTS selfie_base64 TEXT;`

#### `backend/routers/auth.py`
- Add `POST /auth/selfie` — stores base64 in `users.selfie_base64`
- Make `phone` optional in `ProfileUpdateRequest` (default `None`), only update if provided

#### `backend/scraper/routes.py`
- In `_background_scrape()`: after Gmail scrape, run calendar fetch in parallel via `run_in_executor()`
- Use existing `calendar_fetch.build_calendar_service()` + `fetch_events()` + `scraper.db.store_calendar_events()`
- Wrap calendar in try/except — failure doesn't block Gmail completion

#### `backend/routers/queue.py`
- Add `POST /queue/skip/{user_id}` — skip a queued user
- Add `PATCH /queue/reorder` — reorder queue positions
- Add `POST /queue/advance` — mark current active as completed, advance next to active
- Emit `queue_updated` socket event on all queue mutations
- Add 2-min timeout task when user becomes active

#### `backend/main.py`
- On `session_ended`: call queue advance (auto-cycle to next user)
- Add `mirror` socket room — mirror joins on connect
- Emit `queue_updated` to `mirror` room on queue changes
- Add `session_force_end` event handler (from admin)

#### `backend/routers/admin.py` (NEW)
- `GET /admin/queue` — full queue with user names and statuses
- `GET /admin/session` — current session info (user, stats, API calls)
- `GET /admin/stats` — booth stats (total users today, avg session length, etc.)

### Frontend

#### `frontend/src/components/phone/SelfieCapture.tsx` (NEW)
- Camera `<video>` preview + "Take Photo" button
- File upload fallback if camera denied
- Canvas resize to 512x512 JPEG (quality 0.8)
- `onCapture(base64: string)` callback + "Retake" option

#### `frontend/src/app/phone/page.tsx` (REWRITE)
New state machine: `signin → questionnaire → queue → idle → recap`
- **Returning user check**: `localStorage` for `user_id` → verify via API → skip to queue
- **signin**: Name + SelfieCapture + GoogleSignIn → triggers scrape → questionnaire
- **questionnaire**: Current onboarding form (gender, brands, styles, occasions, price) → join queue
- **queue**: QueueStatus component with `onBecameActive` callback → idle
- **idle**: "You're at the mirror!" + cycling tips + ring timer. Listens for `session_ended` → recap
- **recap**: Summary + liked items + "View on Poke" button (URL TBD) + "Done"
- Use Tailwind + shadcn/ui (replace all inline styles)

#### `frontend/src/components/phone/QueueStatus.tsx`
- Add `onBecameActive?: () => void` callback prop
- Fire once via `useRef` when `queue.status === "active"`

#### `frontend/src/lib/api.ts`
- Add `uploadSelfie(userId, selfieBase64)`
- Make `phone` optional in `updateProfile()`
- Add admin API functions: `getAdminQueue()`, `skipQueueUser()`, `reorderQueue()`, `forceEndSession()`, `getBoothStats()`

#### `frontend/src/app/mirror/page.tsx` (MODIFY)
Add kiosk cycling:
- **attract state**: QR code + branding, listen for `queue_updated` socket event
- **waiting state**: "Waiting for [Name]..." + Start/Skip buttons + 2-min countdown
- **session state**: Existing behavior (no changes)
- Remove `user_id` URL param requirement — mirror gets active user from queue
- Join `mirror` socket room on mount

#### `frontend/src/app/admin/page.tsx` (NEW)
Simple admin panel:
- Queue list with skip/reorder controls
- Current session: user info + stats + "Force End" button
- Booth stats: total users, avg session length, items shown/liked
- Auto-refreshes via polling or socket events
