# Plan: Phone Number Authentication for Poke MCP Integration

## Context

Poke's MCP tools (`get_past_sessions` and `save_session`) use phone numbers as the primary user identifier to look up session history. Currently, Mirrorless has a critical gap:

**The Problem:**
1. Phone numbers are not collected during onboarding (no UI field exists)
2. The `users.phone` column exists but is NULL for all users
3. Sessions are created with `user_id` (UUID), but Poke needs `phone` to query them
4. No bridge exists between `session_id` → `phone` for Poke's `save_session` calls

**Why This Matters:**
- Without phone collection, Poke cannot look up past sessions
- Without a session-to-phone bridge, Poke cannot save session summaries
- Users cannot get personalized recommendations based on their Mirrorless history

---

## Solution Approach

### 1. Collect Phone Numbers During Onboarding

Add phone input field to `SignInView` (alongside name field) with real-time E.164 validation.

**Why SignInView?**
- Collects phone early, ensuring it's available before session creation
- Follows existing pattern (name collection happens in SignInView)
- Phone becomes available immediately after OAuth for Poke's needs

### 2. Create Session-Phone Bridge for Poke

Add new endpoint `GET /api/sessions/{session_id}/info` that returns user's phone from session context.

**Data Flow:**
```
Poke receives session_id from mirror event
  → Calls GET /api/sessions/{session_id}/info
  → Backend: SELECT users.phone FROM sessions JOIN users...
  → Returns {session_id, user_id, phone, name, status}
  → Poke uses phone to call save_session(phone, session_id, summary)
```

### 3. Validate Phone Numbers (E.164 Format)

- **Frontend**: `libphonenumber-js` for real-time validation as user types
- **Backend**: `phonenumbers` library to validate on profile update
- **Format**: E.164 standard (`+15551234567`)

```python
import phonenumbers
from phonenumbers import NumberParseException

@router.post("/profile")
async def update_profile(body: ProfileUpdateRequest):
    """Update user name and optionally phone number."""
    db = NeonHTTPClient()
    try:
        # Validate phone if provided
        if body.phone is not None:
            if not body.phone.strip():
                raise HTTPException(status_code=400, detail="Phone number cannot be empty")

            try:
                parsed = phonenumbers.parse(body.phone, "US")
                if not phonenumbers.is_valid_number(parsed):
                    raise HTTPException(status_code=400, detail="Invalid phone number")

                # Normalize to E.164 format
                e164_phone = phonenumbers.format_number(parsed, phonenumbers.PhoneNumberFormat.E164)
            except NumberParseException:
                raise HTTPException(status_code=400, detail="Invalid phone number format")

            rows = await db.execute(
                """UPDATE users SET name = $1, phone = $2
                   WHERE id = $3::uuid
                   RETURNING id, name, email, phone, poke_id""",
                [body.name, e164_phone, body.user_id],
            )
        else:
            # Existing logic for name-only update
            rows = await db.execute(...)

        if not rows:
            raise HTTPException(status_code=404, detail="User not found")
        return rows[0]
    finally:
        await db.close()
```

### Phase 3: Session-Phone Bridge for Poke

**3.1 Create Sessions Router**

File: `backend/routers/sessions.py` (new)

```python
from fastapi import APIRouter, HTTPException
from models.database import NeonHTTPClient
import uuid

router = APIRouter(prefix="/api/sessions", tags=["sessions"])

@router.get("/{session_id}/info")
async def get_session_info(session_id: str):
    """Get session info including user's phone number.

    Used by Poke to map session_id → phone for save_session calls.
    """
    db = NeonHTTPClient()
    try:
        # Validate session_id is valid UUID
        try:
            uuid.UUID(session_id)
        except ValueError:
            raise HTTPException(status_code=400, detail="Invalid session ID format")

        # Join sessions with users to get phone
        rows = await db.execute(
            """
            SELECT
                s.id as session_id,
                s.user_id,
                u.phone,
                u.name,
                s.started_at,
                s.ended_at,
                s.status
            FROM sessions s
            JOIN users u ON s.user_id = u.id
            WHERE s.id = $1::uuid
            """,
            [session_id]
        )

        if not rows:
            raise HTTPException(status_code=404, detail="Session not found")

        return rows[0]
    finally:
        await db.close()
```

## Edge Cases Handled

1. **Missing phone (existing users)**: Allow NULL phone, MCP tools return error
2. **Invalid phone format**: Frontend + backend validation prevents storage
3. **Duplicate phone numbers**: No unique constraint initially (can add later)
4. **International numbers**: `libphonenumber-js` supports all country codes
5. **Session created before phone added**: Session info endpoint returns `phone: null`
6. **Phone format changes**: Sessions reference `user_id` FK (immutable), phone is just a lookup key
