# Plan: Graceful Session Ending with Recap Screen

## Context

When Mira hits the `SOFT_API_LIMIT` (20 Claude API calls), she silently stops responding — the user gets no feedback, no goodbye, no recap. The session stays "alive" but dead. This change makes Mira gracefully wrap up: she delivers a spoken closing recap via TTS, saves the session, and the mirror transitions to a visual recap screen showing liked items and stats before advancing to the next user.

## Changes

### 1. Backend: `backend/agent/orchestrator.py`

**Replace silent API limit with graceful shutdown** (line 272-274):

```python
# Before:
if session.api_calls >= SOFT_API_LIMIT:
    print(f"[mira] API limit reached for {session.user_id}")
    return

# After:
if session.api_calls >= SOFT_API_LIMIT:
    print(f"[mira] API limit reached for {session.user_id}, initiating graceful shutdown")
    if not session.wrapping_up:
        session.wrapping_up = True
        await self._graceful_shutdown(session)
    return
```

**Add `wrapping_up` guard in `handle_event()`** (after line 196):
- If `session.wrapping_up` is True, ignore all incoming events (voice, gesture, etc.) and return early
- This prevents new transcripts from queueing up during the closing speech

**Add `_graceful_shutdown()` method** (new, after `_call_claude`):
1. Cancel the silence timer
2. Call `_generate_closing_speech(session)` — a new Haiku call that generates a warm, in-character Mira closing message (2-4 sentences, with `[emotion:proud]` tag)
3. Stream the closing speech via `_stream_text()` chunks + end-of-message — this triggers TTS on the frontend so the user *hears* the recap
4. `await asyncio.sleep(0.5)` — brief pause to let TTS start
5. Call `self.end_session(user_id)` — saves to DB, emits `session_ended` with recap payload

**Add `_generate_closing_speech()` method** (new, after `_generate_summary`):
- Separate from `_generate_summary()` because: summary = clinical DB record, closing speech = warm spoken goodbye
- Uses Haiku, max 200 tokens
- Prompt: "You are Mira wrapping up a session. Recap liked items, give a confidence boost, tell them picks are saved to their phone. Start with [emotion:proud]. Keep it conversational — no markdown."
- Input: session stats + liked item names
- Fallback on exception: hardcoded "That was a great session! Your favorites are saved to your phone. See you next time!"

**Add `user_name` to `session_ended` payload** (line 446-458):
- Add `"user_name": session.user_context.get("profile", {}).get("name", "")` to the emitted data so the recap screen can greet the user by name

## Event Flow (End-to-End)

```
1. api_calls hits 20 → _graceful_shutdown() fires
2. Silence timer cancelled, wrapping_up = true
3. _generate_closing_speech() → Haiku generates warm recap
4. _stream_text() chunks → mira_speech events → frontend TTS plays through Orb
5. end_session() → saves summary to DB → emits session_ended with full payload
6. Mirror: receives session_ended → shows SessionRecap overlay (summary + items + stats)
7. Mirror: 20s countdown → auto-dismiss → attract state
8. Phone: receives session_ended → shows RecapView (already exists)
9. Backend: _auto_advance_queue() promotes next user → queue_updated fires
```

## Edge Cases

- **User speaking when limit hits**: `wrapping_up` guard in `handle_event()` ignores new events
- **No liked items**: Closing speech handles gracefully ("We explored a lot of styles"), recap hides "Your Favorites" section
- **Closing speech generation fails**: Hardcoded fallback message used
- **TTS still playing when recap appears**: Recap renders as overlay while TTS finishes — no conflict
- **Force-end/disconnect**: These skip the closing speech (already abrupt by design), but still emit `session_ended` so recap screen still shows if data is available
