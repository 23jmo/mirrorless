# Plan: Add `take_photo` Tool for Mira

## Context

Mira's prompt tells her to say "Let me see what you're working with today" to transition from Phase 1 (roast) to Phase 2 (outfit check). But there's no tool for her to actually capture a photo. The current approach is a **hardcoded 10-second delayed snapshot** after session start — and even that is broken due to a **format mismatch bug** where the frontend sends snapshot data in a format the backend silently drops. Result: Mira says she'll look at what you're wearing, and then the session stalls forever.

**Root cause:** Frontend sends `{type: "snapshot", image_base64, user_id}` but `main.py` expects `{user_id, event: {type: "snapshot", ...}}`. The `data.get("event", {})` returns `{}` (falsy) → early return → snapshot never reaches the orchestrator.

## Changes

### 1. Add `take_photo` tool definition
**File:** `backend/agent/tools.py` — Add to `TOOL_DEFINITIONS` array

- No input params needed (just triggers the mirror camera)
- Description tells Claude it's once-per-session for the Phase 1→2 transition
- NOT handled in `execute_tool()` — requires Socket.io + session state only available in the orchestrator

### 2. Add `_photo_taken` and `_snapshot_future` to `SessionState`
**File:** `backend/agent/orchestrator.py` (line 50-69)

- `_snapshot_future: asyncio.Future | None = None` — pending Future when waiting for mirror response
- `_photo_taken: bool = False` — enforces once-per-session

### 3. Handle `take_photo` in `_handle_tool_calls()`
**File:** `backend/agent/orchestrator.py` (line 440-496)

Intercept `take_photo` before calling `execute_tool()`. New method `_handle_take_photo()`:
1. Check `_photo_taken` flag — return error if already used
2. Create `asyncio.Future` on the session
3. Emit `request_snapshot` to the mirror via Socket.io
4. `await asyncio.wait_for(future, timeout=5.0)`
5. Return image as content blocks in tool result (text + image block — Claude API supports this in tool results)
6. On timeout: return error message telling Claude to proceed without the photo

### 4. Resolve the Future in `handle_event()` for snapshots
**File:** `backend/agent/orchestrator.py` (line 196+)

At the top of `handle_event()`, before acquiring `_event_lock`:
- If event is `snapshot` AND `_snapshot_future` is pending → resolve the Future with `image_base64` and return early
- This avoids deadlock since the lock is held by the `_handle_take_photo` caller chain

If no Future is pending, fall through to existing behavior (backward compat).

### 5. Fix snapshot emit format in frontend
**File:** `frontend/src/app/mirror-v2/page.tsx` (line 489-493)
**File:** `frontend/src/app/mirror/page.tsx` (line 365-369)

Change from:
```typescript
socket.emit("mirror_event", { type: "snapshot", image_base64, user_id });
```
To:
```typescript
socket.emit("mirror_event", { user_id, event: { type: "snapshot", image_base64 } });
```
Matches the `{user_id, event: {...}}` format used by voice/gesture events.

### 6. Add backward-compat format handling in `main.py`
**File:** `backend/main.py` (line 406-422)

Before the `if not event` early return, detect legacy top-level format and wrap it:
```python
if not event and data.get("type") == "snapshot":
    event = {"type": "snapshot", "image_base64": data.get("image_base64", "")}
```

### 7. Remove hardcoded 10-second delayed snapshot
**File:** `backend/agent/orchestrator.py` (line 182-192)

Delete the `_delayed_snapshot()` function and its `asyncio.create_task` call. The `take_photo` tool replaces this.
