# Plan: Fix first ~3 TTS messages getting cut off at session start

## Context

At the beginning of each mirror session, the first ~3 Mira TTS responses get cut off (audio barely plays before being interrupted). After that, TTS works fine. There is no explicit "3-attempt limit" — the behavior is caused by **rapid-fire messages at session start overwhelming the barge-in mechanism**.

### Root Cause

The `StreamingTTS.speak()` method uses a **generation counter** for barge-in: each call to `speak()` increments the counter, and any in-flight audio with a stale generation aborts immediately (`streaming-tts.ts:33-46`).

At session start, 3 Claude responses fire in quick succession:

1. **Greeting** (immediate) — `session_start` event → Claude responds → `mira.speak(greeting)` starts TTS fetch
2. **Snapshot response** (~1-2s later) — `request_snapshot` is emitted right after greeting (`orchestrator.py:183`), mirror captures frame → backend sends to Claude → Claude responds → `mira.speak()` increments generation → **greeting audio aborts**
3. **Silence nudge** (5s later) — silence timer fires → Claude responds → `mira.speak()` increments generation → **snapshot response audio aborts**

After this initial burst, user input drives the pace (one response at a time), so the barge-in mechanism never triggers incorrectly.

### Why there's no queue

`useMiraVideoAvatar.speak()` (`useMiraVideoAvatar.ts:60-99`) calls `ttsRef.current.speak()` as fire-and-forget (`.catch()` only). It doesn't await the previous message finishing. So when a new message arrives, it immediately starts a new `speak()`, killing the previous one via the generation counter.

## Fix: Delay the snapshot request at session start

The snapshot request fires at `orchestrator.py:183` immediately after the greeting `handle_event` returns. This causes the snapshot response to overlap with the greeting TTS. By delaying the snapshot, the greeting plays uninterrupted, and the silence timer (5s) has time to fire naturally without also competing with a snapshot response.

### Changes

#### `backend/agent/orchestrator.py` — line ~182-184

Replace the immediate snapshot emission:
```python
# Request a snapshot from the mirror so Mira can see the user
if self.sio:
    await self.sio.emit("request_snapshot", {"user_id": user_id}, room=user_id)
```

With a delayed background task:
```python
# Request a snapshot after a delay so it doesn't overlap with the greeting TTS
async def _delayed_snapshot():
    await asyncio.sleep(10)
    if user_id in self.sessions and self.sessions[user_id].is_active:
        await self.sio.emit("request_snapshot", {"user_id": user_id}, room=user_id)

if self.sio:
    asyncio.create_task(_delayed_snapshot())
```

The 10s delay gives the greeting ~8-10s to stream from Claude + play via TTS. The snapshot then arrives as a natural second interaction rather than a rapid-fire overlap.
