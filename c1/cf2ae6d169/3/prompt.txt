# Fix: Backend responds but frontend never plays voice/avatar

## Context

The backend IS generating Claude responses — server logs show `[mira] AGENT SAID: ...` with full text. But the frontend plays no voice and shows no avatar lip sync. The browser console shows zero `mira_speech`-related logs (no `[Mirror] Agent full response:`, no `[SentenceBuffer]`, no `[TTS]`).

## Root cause: `!data.text` guard kills end-of-message signal

**File:** `frontend/src/app/mirror/page.tsx`, line 128

```typescript
const handleSpeech = (data: { text?: string; is_chunk?: boolean }) => {
  // ...fallback session detection...

  if (!data.text) return;          // ← LINE 128: "" is falsy → exits!

  if (data.is_chunk !== false) {
    responseAccumulatorRef.current += data.text;     // chunks accumulate fine
    avatar.setAvatarState("thinking");
  } else {
    // End-of-message — flush to TTS                 ← NEVER REACHED
    const fullText = responseAccumulatorRef.current;
    sentenceBuffer.feed(fullText);
    sentenceBuffer.flush();
  }
};
```

The backend sends two kinds of `mira_speech` events:
- **Chunks**: `{"text": "some words", "is_chunk": true}` — pass the guard fine
- **End-of-message**: `{"text": "", "is_chunk": false}` — `""` is falsy in JS → dropped

All chunks accumulate silently in `responseAccumulatorRef`, but the end-of-message signal that triggers `sentenceBuffer.feed()` → TTS → avatar is completely dropped.
