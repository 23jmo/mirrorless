# Fix: AvatarPiP container ref null + add logging

## Context

After the HeyGen → Memoji migration, clicking "Start Session" logs `[MemojiAvatar] No container ref — cannot start session`. The avatar never initializes.

**Root cause**: React conditional-rendering + ref timing bug.

```
handleSessionActive():
  setSessionActive(true)        ← state queued, NOT yet rendered
  avatar.startSession()         ← runs NOW, containerRef.current is null

JSX:
  {sessionActive && <AvatarPiP containerRef={...} />}   ← mounts NEXT render
```

`startSession()` needs the container div from AvatarPiP, but that div only mounts after `sessionActive` becomes true on the next React render cycle. By then `startSession()` has already bailed.

## Fix

### 1. `frontend/src/components/mirror/AvatarPiP.tsx`
- Add `visible` boolean prop
- Always render the component (keep ref alive), hide with `display: "none"` when `visible` is false

### 2. `frontend/src/app/mirror/page.tsx`
- Always render `<AvatarPiP>` (remove `{sessionActive && ...}` conditional)
- Pass `visible={sessionActive}` instead

### 3. `frontend/src/hooks/useMemojiAvatar.ts`
- Add more console logs for debugging:
  - `startSession()`: log container ref state, init success/failure
  - `processQueue()`: log queue length and current sentence
  - `playScripted()`: log video path
  - `stopSession()`: log cleanup

## Verification
- `npx tsc --noEmit` — no type errors
- Open `/mirror?user_id=...`, click Start Session
- Console: `[MemojiAvatar] Session started` (no "No container ref" error)
- Idle loop video visible in PiP corner
