# Replace HeyGen LiveAvatar with Jenny Memoji Avatar System

## Context

HeyGen LiveAvatar SDK is broken (developer unresponsive) and being scrapped. The `jenny/` directory contains a working memoji avatar system with pre-recorded video loops, ElevenLabs TTS, and scripted response videos with baked-in audio. This plan merges Jenny's avatar tech into the Mira mirror app.

**What stays**: Deepgram STT, Claude orchestrator, Socket.io, SentenceBuffer
**What goes**: HeyGen SDK, `useLiveAvatar.ts`, `/api/heygen/token` endpoint
**What's new**: Memoji video avatar, ElevenLabs TTS via backend proxy, scripted response matching

---

## New Data Flow

```
Claude streams text → mira_speech socket events
    → Mirror page accumulates full response
    → findScriptedResponse(fullText)
        ├─ MATCH → play scripted MP4 (baked-in audio, perfect lip sync)
        └─ NO MATCH → SentenceBuffer → avatar.speak(sentence)
                        → POST /api/tts/speak (ElevenLabs proxy)
                        → play audio blob + talking_loop.mp4
                        → on ended → idle_loop.mp4 → next sentence
```

## Phase 2: Backend TTS Proxy

**Create `backend/routers/tts.py`** — new endpoint to proxy ElevenLabs API

```
POST /api/tts/speak
Body: { "text": "sentence to speak" }
Returns: audio/mpeg binary (StreamingResponse)
```

- Uses `ELEVENLABS_API_KEY` and `ELEVENLABS_VOICE_ID` from backend `.env`
- Same voice settings as Jenny: model `eleven_multilingual_v2`, stability 0.5, similarity_boost 0.8, style 0.5, speaker_boost true
- Max 1000 chars per request (cost control)
- 500 if key not configured, 502 on upstream error

**Modify `backend/main.py`** — register the new router:
- Add `from routers import tts`
- Add `app.include_router(tts.router)`

## Phase 3: Frontend Core Modules (3 new files)

### 3a: `frontend/src/lib/memoji-avatar.ts`
TypeScript port of Jenny's `MiraAvatar` class (`jenny/src/avatar.js`).

- Manages 6 looping muted `<video>` elements + 1 scripted (unmuted) `<video>`
- States: idle, thinking, talking, happy (5s revert), excited (5s revert), concerned
- Takes a `container: HTMLDivElement` in constructor (injected by React ref)
- `init()` creates videos, preloads, shows idle
- `setState(state)` hides current video, shows+plays new one
- `playScriptedVideo(path): Promise<void>` plays one-shot video with audio, resolves on ended
- `destroy()` pauses all videos, removes from DOM (React cleanup)
- Video paths: `/avatar/loops/{state}.mp4` and `/avatar/scripted/{name}.mp4`

### 3b: `frontend/src/lib/scripted-responses.ts`
Direct TypeScript port of Jenny's `scripted-responses.js`.

- `findScriptedResponse(text): ScriptedResponse | null` — two-tier matching: exact phrase includes, then keyword scoring (threshold 4)
- `detectEmotion(text): AvatarState` — regex-based emotion detection for fallback
- 13 scripted response mappings with video paths pointing to `/avatar/scripted/`
- Export types: `ScriptedResponse`, `AvatarState`

### 3c: `frontend/src/lib/elevenlabs-tts.ts`
TTS client that calls the backend proxy.

- `speak(text, onStart?, onEnd?): Promise<void>` — POST to `/api/tts/speak`, gets audio blob, plays via `Audio` element
- `stop()` — pause and clear current audio
- Revoke blob URLs on end/error to prevent memory leaks
- Browser TTS fallback via `SpeechSynthesisUtterance` if backend proxy fails

## Phase 4: New React Hook

**Create `frontend/src/hooks/useMemojiAvatar.ts`** — replaces `useLiveAvatar`

Public API (mirrors existing interface shape):
```typescript
interface UseMemojiAvatarReturn {
  isReady: boolean;
  isSpeaking: boolean;
  startSession: () => Promise<void>;
  stopSession: () => Promise<void>;
  speak: (text: string) => void;       // queue sentence for TTS + talking loop
  playScripted: (videoPath: string) => Promise<void>;  // NEW: play scripted video
  interrupt: () => void;
  clearQueue: () => void;
  containerRef: RefObject<HTMLDivElement | null>;  // changed from avatarRef/HTMLVideoElement
}
```

Internal: same speech queue pattern as current `useLiveAvatar` — `speechQueueRef`, `isProcessingQueueRef`, `processQueue()`.

- `startSession()`: create `MemojiAvatar(container)` + `ElevenLabsTTS(apiUrl)`, call `avatar.init()`, set ready
- `speak(text)`: push to queue → `processQueue()` → `tts.speak(sentence, avatar.talking, avatar.idle)` → on ended process next
- `playScripted(path)`: clear queue, stop TTS, call `avatar.playScriptedVideo(path)`, set isSpeaking during
- `interrupt()`: stop TTS, avatar.idle(), clear queue
- `stopSession()`: destroy avatar, stop TTS, clear queue

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Accumulating full response adds latency before speech | Set avatar to "thinking" state for visual feedback. Mira's max_tokens=400 so delay is ~2-4s |
| ElevenLabs TTS latency per sentence (~500ms-1s) | Speech queue processes serially; consider audio pre-fetch optimization later |
| Browser autoplay blocks scripted video audio | AudioContext unlock on Start Session click already in place; scripted videos play after user gesture chain |
| ~5.7MB of video assets in public/ | Acceptable for kiosk/TV display. Could lazy-load scripted videos later |
