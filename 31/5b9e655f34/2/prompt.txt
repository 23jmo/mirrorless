# Live Scraper Debug Tool

## What this does

Creates a CLI script that **actually re-scrapes your Gmail inbox live** and prints everything it finds in real-time to the terminal. This is NOT reading from the database — it makes real Gmail API calls, fetches real emails, and runs them through the full extraction pipeline while showing you every step.

## How to run

```bash
cd backend && pip install tabulate && python scrape_debug.py <user_id>
```

Optional flags:
- `--no-llm` — skip Claude API calls (faster, free)
- `--max-per-query 50` — fetch more emails per query (default 20)

## What it prints (streamed as it runs)

### Step 1: Gmail Search (live API calls)
Runs each of the 7 receipt search queries against your actual Gmail inbox and prints results as they come back:


## Implementation

**New file:** `backend/scrape_debug.py` (~300 lines)
**Modified:** `backend/requirements.txt` (add `tabulate`)

The script:
1. Fetches your stored OAuth token from Neon DB (async, same as production)
2. Builds a Gmail API service (same `build_gmail_service()` from `scraper/gmail_auth.py`)
3. Calls `search_emails()` and `get_message_content()` live against Gmail
4. Runs the receipt filter + extraction logic with debug prints at each step
5. Prints summary table and style profile
6. Does **NOT** write anything to the database — purely read + display

Imports directly from existing scraper modules — no logic duplication except for the `_fashion_signals` tuple (which is inline in `extract_purchases()` and can't be imported).
