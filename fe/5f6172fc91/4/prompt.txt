# Plan: Add Session Ending Mechanisms

## Context

Sessions currently have no natural ending — they only stop when the API call limit is hit (20 calls), admin force-ends, or socket disconnects. Users can't say "I'm done" and leave, and there's no end button. This plan adds both: a Mira `end_session` tool (so Claude can end the session when the conversation naturally concludes) and a manual "End Session" button on the mirror UI.

---

## Data Flow

**Tool path** (Mira ends session):
```
Claude streams goodbye text → emits end_session tool_use → _handle_tool_calls detects it
→ sets flag, breaks loop → self.end_session() (summary + DB save + emit session_ended)
→ self._advance_queue() → frontend transitions to recap
```

**Button path** (manual end):
```
Click → socket.emit("end_session") → main.py handler → mira.end_session()
→ _auto_advance_queue() → frontend transitions to recap
```

Both converge on the existing `session_ended` socket event and `SessionRecap` component.
