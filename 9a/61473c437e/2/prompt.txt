# Plan: Fix queue join ON CONFLICT referencing non-existent constraint

## Context

`/queue/join` crashes with: `constraint "uq_queue_user_active_waiting" for table "queue" does not exist`.

The unique index `uq_queue_user_active_waiting` exists (created via `CREATE UNIQUE INDEX`), but `ON CONFLICT ON CONSTRAINT` only works with **constraints** (created via `ALTER TABLE ADD CONSTRAINT`). In PostgreSQL, a unique index and a unique constraint are different catalog objects.

## Fix

### `backend/routers/queue.py` line ~107

Change:
```sql
ON CONFLICT ON CONSTRAINT uq_queue_user_active_waiting DO NOTHING
```

To:
```sql
ON CONFLICT (user_id) WHERE status IN ('waiting', 'active') DO NOTHING
```

This matches the index by its column expression + WHERE clause instead of by name. Works with both unique indexes and constraints.
