# Mirror Experience Wiring — Implementation Plan

## Context

The Mira agent, gesture detection, and Socket.io architecture are all working. But the mirror page is currently just a webcam feed with gesture indicators — there's no voice input, no avatar output, and no product display. This plan wires everything together: HeyGen avatar for Mira's visual/voice output, Deepgram STT for user voice input, a product carousel for clothing recommendations, and real gesture interaction with the carousel.

Full spec: `docs/plans/2026-02-14-mirror-wiring-spec.md`

## Key Design Decisions (from interview)

- **HeyGen**: Streaming Avatar SDK, `TaskType.REPEAT` mode (we send text, it speaks). NOT `startVoiceChat` (that uses GPT-4o, bypasses Claude/Mira).
- **Deepgram**: Browser-direct WebSocket (no backend proxy). Key as `NEXT_PUBLIC_DEEPGRAM_API_KEY`.
- **Speech buffering**: Sentence-boundary buffering (accumulate chunks until `.` `!` `?`) before calling `avatar.speak()`.
- **Mic policy**: Always-on with VAD, but transcripts **queued** while Mira is speaking (no interruption, no lost input).
- **Avatar position**: Top-right PiP corner.
- **Product display**: Bottom carousel, Tinder-style swipe animations.
- **Gesture routing**: Dual — frontend animates cards immediately, backend gets gesture async.
- **Avatar lifecycle**: Created on `start_session`, destroyed on session end (clean reset).
- **HeyGen auth**: Backend endpoint generates session token (key stays server-side).

## Implementation Steps

### Step 1: Backend — HeyGen token endpoint

**Create `backend/routers/heygen.py`**

```python
import os
import httpx
from fastapi import APIRouter, HTTPException

router = APIRouter(prefix="/api/heygen", tags=["heygen"])

@router.get("/token")
async def get_heygen_token():
    api_key = os.environ.get("HEYGEN_API_KEY")
    if not api_key:
        raise HTTPException(status_code=500, detail="HEYGEN_API_KEY not configured")
    async with httpx.AsyncClient() as client:
        resp = await client.post(
            "https://api.heygen.com/v1/streaming.create_token",
            headers={"x-api-key": api_key},
        )
    if resp.status_code != 200:
        raise HTTPException(status_code=502, detail=f"HeyGen API error: {resp.text}")
    return {"token": resp.json()["data"]["token"]}
```

### Step 2: Backend — Add speech-end signal to orchestrator

**Modify `backend/agent/orchestrator.py`**

Currently `_stream_text` emits `mira_speech` with `is_chunk: True` for every chunk, but never signals "done". The sentence buffer needs a flush signal.

After the streaming loop in `_call_claude` (line ~287, after the `async for event in stream:` block), emit one final event:

```python
# After streaming complete, signal end so frontend flushes sentence buffer
await self._stream_text_end(session.user_id)
```

Add method:
```python
async def _stream_text_end(self, user_id: str) -> None:
    if self.sio:
        await self.sio.emit(
            "mira_speech",
            {"text": "", "is_chunk": False},
            room=user_id,
        )
```

This uses the existing `mira_speech` event with `is_chunk: False` to signal the end — no new event type needed.

### Step 4: Frontend — Sentence buffer utility

**Create `frontend/src/lib/sentence-buffer.ts`**

Pure utility class, no React dependencies. Accumulates text chunks and flushes on sentence boundaries (`.` `!` `?` followed by space or end-of-input).

Edge cases to handle:
- Decimal numbers (`$29.99`) — don't split on the decimal point
- Ellipsis (`...`) — don't split on each dot
- Empty chunks — ignore gracefully
- `flush()` — force-send remaining buffer (called on `is_chunk: false`)

### Step 5: Frontend — HeyGen avatar hook

**Create `frontend/src/hooks/useHeyGenAvatar.ts`**

```typescript
interface UseHeyGenAvatarReturn {
  isReady: boolean;
  isSpeaking: boolean;
  startAvatar: () => Promise<void>;
  stopAvatar: () => Promise<void>;
  speak: (text: string) => Promise<void>;
  interrupt: () => Promise<void>;
  videoRef: RefObject<HTMLVideoElement>;
}
```

Lifecycle:
1. `startAvatar()` → fetch token from `GET /api/heygen/token` → create `StreamingAvatar` → `createStartAvatar()`
2. Listen for `STREAM_READY` → attach MediaStream to videoRef → set `isReady = true`
3. Listen for `AVATAR_START_TALKING` / `AVATAR_STOP_TALKING` → update `isSpeaking`
4. `speak(text)` → `avatar.speak({ text, task_type: TaskType.REPEAT })`
5. `stopAvatar()` → `avatar.stopAvatar()` → cleanup

Config: `avatarName` from `NEXT_PUBLIC_HEYGEN_AVATAR_ID` or HeyGen default. Quality: Medium. `activityIdleTimeout: 3600`.

### Step 7: Frontend — Deepgram STT hook

**Create `frontend/src/hooks/useDeepgramSTT.ts`**

```typescript
interface UseDeepgramSTTReturn {
  isListening: boolean;
  transcript: string;
  interimTranscript: string;
  startListening: () => void;
  stopListening: () => void;
}
```

Implementation:
1. `startListening()` → `getUserMedia({ audio: true })` (separate from camera stream)
2. Open WebSocket to `wss://api.deepgram.com/v1/listen?model=nova-2&smart_format=true&interim_results=true&vad_events=true&utterance_end_ms=1500&encoding=linear16&sample_rate=16000`
3. Auth: `token` param with `NEXT_PUBLIC_DEEPGRAM_API_KEY`
4. Use `AudioContext` + `ScriptProcessorNode` (or `MediaStreamAudioSourceNode` + `AudioWorkletNode`) to capture PCM16 audio frames → send as binary WebSocket messages
5. Parse incoming JSON messages for `transcript` (final) and `speech_final` events
6. On `utterance_end` or final transcript → update `transcript` state
7. On interim results → update `interimTranscript`
8. `stopListening()` → close WebSocket, stop audio stream

Reconnect: one attempt on disconnect. If fails, set `isListening: false`.
