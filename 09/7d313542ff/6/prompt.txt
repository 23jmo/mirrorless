# Migrate from HeyGen Streaming Avatar to LiveAvatar SDK

## Context

The project uses the deprecated `@heygen/streaming-avatar` SDK (old HeyGen API at `api.heygen.com`). HeyGen has moved to **LiveAvatar** — a separate product with a new SDK (`@heygen/liveavatar-web-sdk`), new API (`api.liveavatar.com`), and event-driven architecture via LiveKit. The old API key returns 401. This migration swaps the SDK, updates the backend token endpoint, and re-integrates the avatar into the stripped-down mirror page.

## Step 2: Rewrite backend token endpoint

**File**: `backend/routers/heygen.py`

Old: `GET /api/heygen/token` → `POST https://api.heygen.com/v1/streaming.create_token`
New: `POST /api/heygen/token` → `POST https://api.liveavatar.com/v1/sessions/token`

Key changes:
- Method: GET → POST (accepts optional `{ is_sandbox: bool }`)
- URL: `api.heygen.com` → `api.liveavatar.com`
- Header: `x-api-key` → `X-API-KEY`
- Request body: must include `mode`, `avatar_id`, `avatar_persona` (from env vars)
- Response: `{ data: { token } }` → `{ session_token, session_id }`
- New env vars: `LIVEAVATAR_AVATAR_ID`, `LIVEAVATAR_VOICE_ID`, `LIVEAVATAR_CONTEXT_ID`

## Step 4: Create `useLiveAvatar.ts` hook

**File**: `frontend/src/hooks/useLiveAvatar.ts`

Public interface (similar shape to old hook):
```typescript
export interface UseLiveAvatarReturn {
  isReady: boolean;
  isSpeaking: boolean;
  startSession: () => Promise<void>;
  stopSession: () => Promise<void>;
  speak: (text: string) => void;    // queues text, calls session.repeat()
  interrupt: () => void;
  clearQueue: () => void;
  avatarRef: React.RefObject<HTMLVideoElement | null>;
}
```

### SDK mapping (old → new)

| Old (`StreamingAvatar`) | New (`LiveAvatarSession`) |
|---|---|
| `new StreamingAvatar({ token })` | `new LiveAvatarSession(sessionToken)` |
| `createStartAvatar({ quality, avatarName })` | `session.start()` |
| `stopAvatar()` | `session.stop()` |
| `speak({ text, taskType: REPEAT })` | `session.repeat(text)` — **sync, returns void** |
| `interrupt()` | `session.interrupt()` — **sync** |
| `videoRef.srcObject = event.detail` | `session.attach(videoElement)` |
| `StreamingEvents.STREAM_READY` | `SessionEvent.SESSION_STREAM_READY` |
| `StreamingEvents.AVATAR_START_TALKING` | `AgentEventsEnum.AVATAR_SPEAK_STARTED` |
| `StreamingEvents.AVATAR_STOP_TALKING` | `AgentEventsEnum.AVATAR_SPEAK_ENDED` |
| `StreamingEvents.STREAM_DISCONNECTED` | `SessionEvent.SESSION_DISCONNECTED` |

### Speech queue
Port the existing one-at-a-time queue pattern from `useHeyGenAvatar.ts`:
- `speak(text)` pushes to queue, calls `processQueue()`
- `processQueue()` shifts next item, calls `session.repeat(next)` (sync, no await)
- On `AVATAR_SPEAK_ENDED`: reset processing flag, call `processQueue()` for next sentence
- `clearQueue()` empties queue + resets flag

### Key detail: `attach()` timing
Call `session.attach(avatarRef.current!)` inside the `SESSION_STREAM_READY` event callback. The `<video>` element must already be in the DOM at that point — guaranteed because `AvatarPiP` renders with `avatarRef` before `startSession()` is called.

### No voiceChat
Don't enable LiveAvatar's built-in `voiceChat` — it routes audio to HeyGen's LLM. We need transcripts going to our Claude-based orchestrator via Deepgram + Socket.io.

## Step 6: Re-integrate mirror page

**File**: `frontend/src/app/mirror/page.tsx`

### Imports to add


### State & hooks
```typescript
const [sessionActive, setSessionActive] = useState(false);
const [isStarting, setIsStarting] = useState(false);
const [products, setProducts] = useState<ProductCard[]>([]);
const avatar = useLiveAvatar();
const stt = useDeepgramSTT();
```

### Session flow
1. **Start button click** → `setIsStarting(true)`, emit `start_session` socket event
2. **`session_active` event** → `setSessionActive(true)`, `setIsStarting(false)`, call `avatar.startSession()`, call `stt.startListening()`
3. **`mira_speech` events** → feed chunks to SentenceBuffer → `avatar.speak(sentence)`
4. **`mira_speech` with `is_chunk: false`** → `sentenceBuffer.flush()`
5. **Deepgram transcripts** → queue while `avatar.isSpeaking`, send via `mirror_event` when avatar quiet
6. **`tool_result` events** → update `products` state for carousel
7. **`session_ended` event** → cleanup: `avatar.stopSession()`, `stt.stopListening()`, reset state

### Fallback session detection
Listen for first `mira_speech` as backup session-start signal (in case `session_active` isn't emitted).

### JSX structure
```
<Suspense fallback={null}>
  <MirrorPage />      ← uses useSearchParams
</Suspense>

Inside MirrorPage:
  - <video> webcam feed (mirrored, full screen)
  - Start Session button (pre-session, centered overlay)
  - <AvatarPiP> (top-right, session only)
  - <GestureIndicator>
  - <ProductCarousel> (bottom, when products exist)
  - <VoiceIndicator> (bottom-left, session only)
  - Error indicators
```
