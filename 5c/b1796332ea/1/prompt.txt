# Plan: Next.js API Routes for Phone Flow (Auth + Queue)

## Context

The phone onboarding flow (Google sign-in â†’ queue â†’ wait) currently calls the Python backend at `NEXT_PUBLIC_API_URL` (`localhost:8000`). This works on the developer's computer but fails on phones accessing `mirrorless.vercel.app` because `localhost:8000` doesn't exist on the phone. The backend is difficult to deploy to Render right now.

**Solution:** Create Next.js API routes (`app/api/*/route.ts`) that run as Vercel serverless functions. These handle auth + queue by connecting directly to Neon Postgres. The mirror AI session continues to use the local backend.

**Architecture split after this change:**
- **Vercel (serverless):** Google OAuth, user CRUD, queue management
- **Local backend:** AI orchestrator, Socket.io, TTS, STT config, admin endpoints, Gmail scraping

### `frontend/src/lib/api.ts` â€” Route auth/queue to Next.js API

Add a second request helper for Next.js API routes (relative paths):
```ts
async function localRequest<T>(path: string, options?: RequestInit): Promise<T> {
  const res = await fetch(path, {
    headers: { "Content-Type": "application/json" },
    ...options,
  });
  if (!res.ok) {
    const body = await res.json().catch(() => ({}));
    throw new Error(body.detail || body.error || `Request failed: ${res.status}`);
  }
  return res.json();
}
```

## Queue Without Socket.io

The phone already polls `GET /queue/status/{userId}` every 5 seconds. It doesn't rely on Socket.io for queue updates. The only Socket.io event the phone listens for is `session_ended` (after the mirror session), which still comes from the local backend when the mirror is running.

The mirror display receives `queue_updated` via Socket.io from the local backend. Since the Vercel API routes can't emit Socket.io events, the mirror won't auto-update when someone joins via phone. **Workaround:** The mirror already polls or the admin can refresh. For the demo, this is acceptable â€” the primary goal is phone sign-in working from any device.
